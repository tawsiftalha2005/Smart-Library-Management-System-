A **console-based Library Management System** developed using **Java** and **Object-Oriented Programming (OOP)** principles. This project simulates the daily operations of a real-world library by allowing administrators to manage books, members, borrowing records, and return transactions through a simple menu-driven interface.

The project is being developed with a strong focus on **clean code**, **modular architecture**, and **maintainability**, making it a valuable portfolio project while strengthening practical Java development skills.

---

# 🎯 Why I Built This Project

As a Computer Science student, I wanted to build a project that goes beyond basic CRUD operations and reflects how object-oriented design can solve real-world problems.

Instead of using a graphical interface or external frameworks, I chose a console-based architecture to focus on writing clean business logic, proper class design, and maintainable code. This project also serves as a practical demonstration of Java fundamentals, software design, and problem-solving skills.

# ✨ Features

## 📚 Book Management

- Add new books
- Update book information
- Delete books
- Search books by ID
- Search books by title
- Display all available books
- Track available copies

## 👥 Member Management

- Register new members
- Update member information
- Remove members
- Search members
- Display all members

## 🔄 Borrow & Return

- Borrow books
- Return books
- Borrow history
- Update available stock automatically
- Prevent borrowing unavailable books

## 💰 Fine Management

- Calculate overdue fines
- Display fine amount
- Maintain return records

## 📂 File Management

- Store books permanently
- Store members permanently
- Store borrowing history
- Load saved data automatically
